# EE871-E2 Release Notes 1.1.0

Date: 2026-06-27
Branch: `hardening/ee871-production-api-20260627`

## Summary

`1.1.0` is a backward-compatible production API hardening release. It keeps the
core synchronous, framework-neutral, allocation-free, and owned by one external
task/context. The release adds optional absent-device startup, stronger identity
validation, checked CO2 sample helpers for average and fast reads, cooperative
long-delay callbacks for persistent-write waits, and matching CLI/example
coverage.

No new hardware HIL pass is claimed for this release. The new behavior is
covered by native fake-transport tests and remains pending for hardware
validation.

## Key Changes

- Added `BeginPolicy::AllowAbsent` so applications can start initialized in
  `OFFLINE` when startup probing sees a presence/transport failure.
- Latched `OFFLINE` for normal tracked operations; they return `BUSY` without
  touching E2 lines until `recover()` succeeds.
- Validated EE871 group, subgroup, and CO2 available-measurement support in
  `begin()`, `probe()`, and `recover()`.
- Added `readCo2AverageSample(Co2ReadResult&)` and
  `readCo2FastSample(Co2ReadResult&)`.
- Added append-only `Err::CO2_SENSOR_ERROR`.
- Added `Config::delayMs`, `Config::yield`, and `Config::longDelaySliceMs` for
  cooperative millisecond write-delay waits.
- Extended cache-only settings with begin policy, startup probe status, delay
  callback presence, and normalized long-delay slice.
- Added Arduino and ESP-IDF CLI commands `sampleavg` and `samplefast`.

## Read Behavior

- `co2avg` / `readCo2Average(uint16_t&)`: raw MV4 value read.
- `co2fast` / `readCo2Fast(uint16_t&)`: raw MV3 value read.
- `sampleavg` / `readCo2AverageSample(Co2ReadResult&)`: MV4 first, status
  second, with status/error/range validation.
- `samplefast` / `readCo2FastSample(Co2ReadResult&)`: MV3 first, status
  second, with status/error/range validation.
- `status` / `readStatus(uint8_t&)`: explicit status-byte read; it can start or
  trigger an EE871 measurement under the documented timing conditions.

## Compatibility

- Existing raw read APIs remain source-compatible.
- Existing `status`, `co2avg`, and `co2fast` example commands remain
  source-compatible in behavior.
- `SettingsSnapshot` layout changed. Rebuild applications and do not persist or
  externally serialize raw snapshot layouts.
- `Err::CO2_SENSOR_ERROR` was appended to avoid renumbering existing errors.
- The core still does not own GPIO, Arduino `Wire`, ESP-IDF I2C handles, tasks,
  locks, or board pin setup.

## Migration From 1.0.0

1. Rebuild applications against `1.1.0`.
2. For field deployments where the probe may be absent at boot, set
   `Config::beginPolicy = BeginPolicy::AllowAbsent` and call `recover()` when
   the application wants to re-probe.
3. Use checked sample helpers for production CO2 values that should be gated by
   the EE871 status byte.
4. Keep raw MV3/MV4 reads for diagnostics, warm-up characterization, or cadence
   policies that intentionally avoid the status read.
5. If a maintenance operation may wait for milliseconds, wire `delayMs` and
   `yield` callbacks when the owner task should cooperate with the platform.
